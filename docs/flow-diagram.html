/import-project-preset Flow

O que acontece quando você roda o importer
1
Escaneia Projeto
Verifica se a pasta tem arquivos de projeto e qual é a stack
pubspec.yaml package.json pyproject.toml go.mod
2
Pasta vazia ou tem código?
Decisão baseada na presença de arquivos de projeto
Pasta vazia
→ /kickoff
7 perguntas → brief → escolhe stack
→ /design-phase
Figma · AI-generated · ou skip
→ apply preset
Scaffolds projeto com preset escolhido
Tem código
→ analisa libs estruturais
Identifica state mgmt, ORM, routing, validation
→ auto-select preset
Match stack detectada com preset
→ diff preview → apply
Mostra o que mudaria · conflitos → .kit-new
3
Apply Preset
Safe write: cria arquivos ausentes · ignora iguais · conflitos → .kit-new
4
Verify Import
Valida que os arquivos essenciais foram aplicados
CLAUDE.md commands/ docs/ai/ roles/
5
Sync Design System
Escaneia tokens de design existentes e sincroniza com DESIGN_SYSTEM.md
Achou tokens → reescreve DESIGN_SYSTEM.md com valores reais
Parcial → preenche o que encontrou
Não achou → pergunta se quer rodar /design-phase
6
Customizar Guides com Libs
Injeta patterns das libs detectadas nos guides de documentação
Riverpod
ARCHITECTURE + CODING_STANDARDS
TanStack Query
FEATURE_GUIDE + ARCHITECTURE
Prisma / SQLAlchemy
DATABASE_GUIDE + ARCHITECTURE
Zod / Pydantic
CODING_STANDARDS + API_GUIDE
GoRouter / React Router
ARCHITECTURE
7
Resposta Final
preset aplicado + libs detectadas + guides customizados + próximo passo sugerido
✓ Projeto pronto para lifecycle
Todos os arquivos do preset foram aplicados, guides customizados com as libs detectadas
/jarvis-plan → implementar → /jarvis-test-flow → /ship
Nenhum preset encontrado? bootstrap-ai create <name> --from "descrição" gera um do zero.
