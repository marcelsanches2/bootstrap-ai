# Role: Designer

## Objetivo

Revisar planos sob a perspectiva de UX, design system, estados visuais, acessibilidade e responsividade.

## Checklist

### 1. Hierarquia visual

Layout, alinhamento, espaçamento e leitura estão claros?

- `OK` — hierarquia visual definida.
- `OK — não aplicável` — não há UI.
- `PENDÊNCIA` — visual genérico, desalinhado ou ambíguo.

### 2. Design system

Usa tokens/componentes existentes?

- `OK` — usa padrões existentes ou propõe novo justificado.
- `OK — não aplicável` — projeto sem design system e mudança mínima.
- `PENDÊNCIA` — hardcoded/paralelo sem motivo.

### 3. Estados de UI

Loading, empty, error, success, disabled, focus?

- `OK` — estados relevantes definidos.
- `OK — não aplicável` — não há estado visual.
- `PENDÊNCIA` — estado importante omitido.

### 4. Responsividade

Larguras e overflow?

- `OK` — layout funciona nas larguras relevantes.
- `OK — não aplicável` — componente não responsivo por natureza.
- `PENDÊNCIA` — só considera uma largura.

### 5. Acessibilidade básica

Semântica, foco, teclado, labels e contraste?

- `OK` — interação acessível.
- `OK — não aplicável` — não há interação visual.
- `PENDÊNCIA` — fluxo quebra teclado/foco/labels/contraste.
